2 # Implementation Plan - Transition from Mocks to Real Database Integrations

This plan outlines the steps required to implement actual PostgreSQL and JSON DevStore database interactions, replace mock endpoints, fix Candidates module compilation errors, and connect the frontend UI components to the API.

## User Review Required

> [!IMPORTANT]
> The TypeScript compiler currently fails because `CandidatesController` calls methods (`getProfileWithUser`, `improveResumeWithAi`) that do not exist on `CandidatesService`, and there are argument mismatches. We will fix these compilation errors first to allow the application to build and run.
>
> For the database integrations, we will support both **PostgreSQL** (production mode) and **JSON DevStore** (development/mock mode enabled by `SKIP_DB_CONNECT=true` or `DISABLE_REDIS=true`).

### Candidates Module (Compilation Fixes)

#### [MODIFY] [candidates.service.ts](file:///d:/startup2/apps/api/src/modules/candidates/candidates.service.ts)
- Inject `AiService` from `AiModule` into `CandidatesService`.
- Implement `getProfileWithUser(userId: string)`:
  - In Dev mode: Load candidate profile and merge with User data from `DevStore`.
  - In Prisma mode: Query candidate profile including user details.
- Implement `improveResumeWithAi(userId: string, resumeId: string)`:
  - Fetch the resume by ID.
  - Call `AiService.improveResume` with the resume's `content` and `userId`.
  - Save the improved resume content back to Prisma/DevStore.
  - Update `aiScore` (e.g., generate a score between 75 and 99).
  - Return the updated resume.
- Fix argument mismatch for `createResume` and `updateResume` calls in `CandidatesController`:
  - Align service signatures and DTO properties (e.g., handling salary structures).

#### [MODIFY] [candidates.controller.ts](file:///d:/startup2/apps/api/src/modules/candidates/candidates.controller.ts)
- Update `createResume` to pass `dto.title` and `dto.content` separately.
- Update `updateResume` to map types from `UpdateResumeDto` to `{ title?: string; content?: ResumeContent }`.
